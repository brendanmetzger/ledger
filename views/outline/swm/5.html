The Document Object Model

In the beginning all the technologies we are using probably seem very independent from one another as you gain experience you will see that they blur together in consistent ways. Let's start with the things we know a little bit about.

Tree Structures

The Document Object Model, or dom, is a standardized, cross-platform and language-independent convention for representing and interacting with objects in html, xhtml and xml documents. The objects, or nodes of every document are organized in a tree structure. Objects in the dom tree may be addressed and manipulated with programming languages (wink/nod JavaScript). The public interface of a dom is specified in its application programming interface (API).
