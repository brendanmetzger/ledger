Selectors: Classes, IDs, and pseudo classes

Now that we are familiar with element selectors and HTML outlining structure, we take our first step towards naming things. Classes and IDs are added as attributes to HTML Elements. Attributes are keywords followed by an = sign and a quoted string of text.

Adding classes to elements is the most powerful way to style a document. Adding IDs is less important for styling and more important for targeting a unique (that means only one) element—which is especially useful in JavaScript and XML. Pseudo-classes are ways to style transient states, such as a user hovering over an element with :hover, or an element’s position in the hierarchy with :first-child They are called pseudo because you don’t add them, the browser kind of adds them for you when necessary.

With well structured HTML, you need fewer elements to accomplish an elegant design through CSS—as opposed to sloppy HTML that mandates too many classes/ids and other selectors, and the less you write, the easier it is to maintain code. Some general rules of thumb that I think works great: if your CSS class doesn’t seem like it could apply to more than one element, you should rethink your approach. If your ID seems like it could apply to several things, you should rethink your approach. Lastly, I think you should steer clear of ID selectors for CSS, only use them to add meaning, uniqueness, and specificity to your markup.
