All about Nodes

Every interval we will add more to our repertoire, building on fundamentals week by week. After an introduction to filesystems and networks, once on a server, a website’s raison d'être is to broadcast information, and that information must be marked up for a machine to parse and organize. This is not a question of style and aesthetic, it is about communication.

Review/Questions

Last week we were introduced to some tools used for web development. I will use Atom in this course. You learned about a few protocols, namely hypetext transfer protocol for using a browser to got files from web servers as well as the file transfer protocol for getting files to and from computers—typically a personal computer and a computer acting as a web server.

Keys to successful web-development

  • Learn about your text editor—watch a tutorial video—they have lots of helpful features.
  • Try to use the mouse less and less every week
  • Learn shortcuts for common tasks (saving files, refreshing browsers, switching between tabs, opening/closing files, etc.)
  • Always have your browser console open when you are developing projects

Sectioning and Outlining
Many developers suffer from a syndrome known as divitis, which is an overuse of <span> and <div> elements. While span and div are handy, and do fulfill a role, conscientious web developers engage in semantic markup, starting with using proper sections and outlines.
