The Nodes

Any website’s raison d'être is to broadcast information, and that information must be marked up for a machine to parse and organize. This is not a question of style and aesthetic, it is about communication. Communicating is actually a difficult art form, much harder than today’s concept: structuring lots of individual pieces into a big idea.

Review & Warm Up

Last week some tools used for web development (servers and text editors) and concepts surrounding filesystems and networks. For tools I will use Brackets and Cyberduck—not because they are the best, but they have useful features I think many of you will appreciate. I actually prefer that you use whatever tools you feel most comfortable with, so if you feel comfortable with something else and it meets our course objectives, then use it.

You learned about a few protocols, namely hypetext transfer protocol for using a browser to got files from web servers as well as the file transfer protocol for getting files to and from computers—typically a personal computer and a computer acting as a web server.

Keys to successful web-development

  • Learn about your text editor—watch a tutorial video—they have lots of helpful features.
  • Try to use the mouse less and less every week
  • Learn shortcuts for common tasks (saving files, refreshing browsers, switching between tabs, opening/closing files, etc.)
  • Always have your browser console open when you are developing projects
