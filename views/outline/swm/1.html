HTML

It's important to consider that you are now marking up a document using a careful hierarchy and structure. This serves two purposes: first to add meaning so that a machine can accurately parse the data, and second, to create nodes that you can find with CSS and apply style and aesthetic.

Sectioning Blocks

<article>
The HTML <article> element represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable (e.g., in syndication). This could be a forum post, a magazine or newspaper article, a blog entry, an object, or any other independent item of content. Each <article> should be identified, typically by including a heading (<h1>-<h6> element) as a child of the <article> element.

CSS: Descendant Selectors and Typography

Declaring CSS

Glancing at an overview of the syntax, the declaration of a CSS ruleset resembles a programming data structure to some degree. Curly braces, colons, semicolons and careful use of punctuation operators is the starting point. From the onset, organization and code conventions are not only important, but mandatory for this course.

Units (of measure)

There are several CSS units, but you really only need to get comfortable with a handful of them.

px (pixels)
em
an em is the width of a letter 'M', it is percentage based, so it is good for responsive design
%
Percentages are tough to get the hang of. They generally work well, and I would recommend them for widths.
vh, vw, vmax, vmin
A percentage of the viewport height and viewport width respectively. This is a handy one.

Text Properties

text-align
Used to align left, center, right align, as well as justify text. You can only align text in block-level elements.
text-transform
Offers transform to upper/lowercase, can capitalize a sentence, etc.
text-decoration
Controls underline, line-through, overline, etc.
text-indent
set the indentation using a CSS unit.

Font Properties

font-size
Changes the size of text in a selected element. Can use any unit.
font-family
Changes the typeface of an element. Specify this as a sequences of typefaces.
font-weight
Controls weight thin/normal/bold, or in 3 digit numbers like 200, 400, 800 (prefferred)
font-variant
Can add small-caps

Review & Extra: Text, Editors, and Keyboards

In this class we are going to use Text Editors; for those unfamiliar, we'll get farther by starting with what a text editor is not. Dreamweaver, Word, Pages, or anything that gives you an option to write anything other than monospaced, non-italicized, single-weight text is not a text editor. Text editors write plain text—text with the sole purpose of representing sequences of characters that make up the necessary syntax of a markup language, stylesheet language, or programming language.

Keys to successful text editing

  • Try to use the mouse less and less
  • Learn shortcuts to jump to front and back of lines
  • Saving a document and moving to a browser and refreshing can be a swift mouse-less motion.
