Nodes into HTML

Your template files is full of boilerplate that might seem like a formality, but it’s important to start understanding that marking up a document using a hierarchy of nodes is about explicity structure and meaning—not about style in the least. This serves two purposes: first to ensure a machine can parse the data, and second, to create objects that you can manipulate. We can change the look and feel of these objects through the use of a stylesheet language, and we can change the behavior of these objects through the use of a programming language. The nodes (html) give structure and proper structure allows for powerful design and behavior modification. Perhaps unfortunately due to it's easiness to learn and write, sloppy and incorrect markup is quite easy to get away with so you should be extra careful to understand it conceptually so you can take advantage of the powerful tools it will provide as we learn more about controlling look/feel and behavior of applications.

The first clue that gives credence to the importance of HTML documents is this boilerplate—something that many developers, at best, neglect, and at worst, never truly learn. Every html file will be saved with a .html extension and will contain the following elements: <!DOCTYPE html>, <html>, <head> and <body>. No other files we author requires such boilerplate; obviously it's there for a reason or we'd skip it! With that, we are going to look at markup from a very high level, and investigate the concept of what a node is, and then apply that concept to authoring documents.

Anatomy of the cascading stylesheet language

There are essentially two things that make a stylesheet work, the first is the selector, and once the selector has found its target, you can begin modify the targets properties. Become familiar with those two terms, I will say selector and property with some frequency. Selector means find this node and properties means adjust the way this thing looks.

Selectors

Selectors are the handles that target specific HTML Element Nodes. Specific means anything from one element such as #special { ...props to literally every element, with something like * { ...props. Selectors can be concatenated with a comma to apply one set of property settings to several selectors—concatenating the previous examples would look like so: #special, * { ...props. Selectors are a heady topic, and one that we will be spending easily the next five weeks on.

Properties

Properties are simply the things we can adjust—these are truly the styles in stylesheet.

Syntax

Stylesheet languages, like Markup Languages, are quite forgiving, which we'll never use to our advantage. A typo in these languages isn't treated as a big deal, but I see it as one.

Typographic Properties

It has been written that Web Design is 95% typography and giving some weight to that claim is the amount of leverage you have for controlling the typographic experience.

font-size property
Font-size sets the size of the text. Other properties can be based off of this size; when you declare a font-size for a particular element, that size will govern how all other properties are set when using a unit such as em
font-family
Changes the typeface of an element. Specify this as a sequences of typefaces.
font-weight
Controls weight thin/normal/bold, or in 3 digit numbers like 200, 400, 800 (preferred, especially for full typefaces with more than three weights)
line-height property (leading)
Usage is line-height: (num)unit;
Unlike print typography dominated by baselines, web typography is dominated by this unit. Mostly, you will use to control leading, or the space between lines.
line-height need not be a fixed number, it can also be a unit-less ratio such as 1.5 which would represent 1.5 times the value of whatever the font-size is—handy!
letter-spacing property (tracking)
text-align
Used to align left, center, right align, as well as justify text. You can only align text in block-level elements.
text-transform
Offers transform to upper/lowercase, can capitalize a sentence, etc.
text-decoration
Controls underline, line-through, overline, etc.
text-indent
set the indentation using a CSS unit.

Javascript & Programming: The briefest introduction.

As humans, have you ever wondered why do we count to 10?. Is there a better way?

The fascinating thing is that there are many ways to count, in fact, an infinite number of ways, and ancient peoples had different methods, usually best suited to their needs and commerce. It wasn't until the 20th century after decades and decades of understanding the principles of boolean algebra that people began realizing a machine could be able to count for us. Modern computers are essentially very complicated clocks; they turn electricity into a trove of circuits that at the most basic level tick billions of times a second and thus, can count pretty fast.
