Transitions

CSS Transitions allow property changes in CSS values to occur smoothly over a specified duration

  • Transitions enhance presentation, they should not provide meaning.
  • Transitions degrade gracefully by default (or they should)
  • Not all properties are animatable (display, for instance); here is the list of animatable properties

Transitions are quite easy to implement. Designing with the full power of transitions can be quite masterful.

Look at the contrasting CSS and Javascript as it illustrates a key point about the nature of our programming languages. First, I haven't changed the structure of my markup one smidge to accommodate the CSS or JavaScript I am using. The next is that CSS is by far most verbose part of any program I write. The last is the JavaScript, when aided by efficient CSS and HTML, can be very concise.

Please Note: I wrote the JavaScript in an extra obscure manner not only to illustrate this point, but so that you don't spend too much time trying to implement something similar. You will learn much easier techniques for accomplishing this task using loops, variables and other beginner-friendly methods.

You'll notice a lot of resources mention vendor-prefixes. At this point, un-prefixed use of the transition property has wide support. Any tutorial or literature using a prefix is somewhat outdated. This is why it is so important to do research and cite multiple resources for information.
