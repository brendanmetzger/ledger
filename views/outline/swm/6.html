Transitions and Transforms

Other than the initial five letters, transitions and transforms have little to do with one another. The reason they might be good to learn at the same time is that potential confusion could arise if you happened to let google auto-suggest take the reins; How do I... CSS... Trans... could send you down the wrong path.

From the W3C spec regarding transitions: CSS transitions allow property changes in CSS values to occur smoothly over a specified duration Essentially, transitions are animating from one style to another—which used to be the purview of fairly complicated JavaScript—but is now very straight forward.

When researching transitions, you'll no doubt notice a lot of resources using vendor-prefixes. At this point, un-prefixed use of the transition property has wide support, and any tutorial or literature using a prefix is somewhat outdated. This is why it is so important to research and cite multiple resources for information.

Transforms, on the other hand, apply methods and vernacular commonly seen in programming, especially in regards to how elements are painted or drawn. From the spec on transforms: CSS transforms allows elements styled with CSS to be transformed in two-dimensional or three-dimensional space.

Keep in Mind

When you learn how to jazz things up it quickly becomes tempting to go a little crazy. Starting out, I say what the hell, go over the top... but I would be remiss if I didn't point out that sometimes effects can get annoying when the pile up or make things difficult to use. In addition to that, I've made a list of a few pointers for the road, if you will.

  • Transitions enhance presentation, they should not provide meaning.
  • Transitions degrade gracefully by default (or they should); ie, nobody should be impacted if they go missing.
  • Not all properties are animatable (display, for instance); here is the list of animatable properties.
  • Transitions are not Animations, which are also possible, but something wholly different.
  • I won't talk further about keyframe animations in this course—you're welcome to explore them on your own.

Intro to DOM Programming

I'm going to expose the tiniest bit of JavaScript that should dovetail nicely with the concepts we’re becoming comfortable with. Namely, we know that when we load up a big HTML file that it it is converted into this very specific object, and this object, like any object, has properties that describe it. Some of the properties actually perform tasks, and as such, they are actually a function of the object, often called methods. Get comfortable with the terms property and method when talking about objects.

To further analyze this way of thought, picture your oven. It has knobs and burners, it has a color, a manufacturer, and it might have properties indicating its state, like clean or dirty. These are all properties, but let's note the differences between how they describe the oven. The color property does not change, the clean property will flip between two values depending on use. The knobs property are where things get interesting, notice that they make the oven do something. Acting on a knob might change the temperature property, it might kick off a process that causes the state to switch from clean to dirty.

As meta as this all seems, it's really quite straightforward. We know we have a document, and that document will tell us things about itself, like when it was last modified: document.lastModified (try typing that in your console and see what happens). Of present concern is the things we can ask the document to do for us, like find elements.

querySelector / querySelectorAll

  1. Open your console
  2. type document.querySelector
  3. type document.querySelector('h2');
  4. type document.querySelectorAll('h2');
  5. type document.querySelectorAll('h2').item(0);
  6. compare #1 to #4

Look at the contrasting CSS and Javascript as it illustrates a key point about the nature of our programming languages. First, I haven't changed the structure of my markup one smidge to accommodate the CSS or JavaScript I am using. The next is that CSS is by far most verbose part of any program I write. The last is the JavaScript, when aided by efficient CSS and HTML, can be very concise.

Please Note: I wrote the JavaScript in an extra obscure manner not only to illustrate this point, but so that you don't spend too much time trying to implement something similar. You will learn much easier techniques for accomplishing this task using loops, variables and other beginner-friendly methods.

When using transforms, be aware that you'll only be able to apply transformations to a transformable elements, that is, an element that applies block-formatting via the box-model. Remember, any element can be forced into a block formatted element explicitly by setting display property to 'block' or 'inline-block', or implicitly when you have set a float or used an absolute or fixed position.
