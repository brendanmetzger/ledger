Transitions and Transforms

Other than the initial five letters, transitions and transforms have little to do with one another. The reason they might be good to learn at the same time is that potential confusion could arise if you happened to let google auto-suggest take the reins; How do I... CSS... Trans... could send you down the wrong path.

From the W3C spec regarding transitions: CSS transitions allow property changes in CSS values to occur smoothly over a specified duration Essentially, transitions are animating from one style to another—which used to be the purview of fairly complicated JavaScript and is now, remarkably easy to accomplish.

When researching transitions, you'll no doubt notice a lot of resources using vendor-prefixes. At this point, un-prefixed use of the transition property has wide support, and any tutorial or literature using a prefix is somewhat outdated. This is why it is so important to research and cite multiple resources for information.

Transforms, on the other hand, apply methods and vernacular commonly seen in programming, especially in regards to how elements are painted or drawn. From the spec on transforms: CSS transforms allows elements styled with CSS to be transformed in two-dimensional or three-dimensional space.

Keep in Mind

When you learn how to jazz things up it quickly becomes tempting to go a little crazy. Starting out, I say what the hell, go over the top... but I would be remiss if I didn't point out that sometimes effects can get annoying when the pile up or make things difficult to use. In addition to that, I've made a list of a few pointers for the road, if you will.

  • Transitions enhance presentation, they should not provide meaning.
  • Transitions degrade gracefully by default (or they should); ie, nobody should be impacted if they go missing.
  • Not all properties are animatable (display, for instance); here is the list of animatable properties.
  • Transitions are not Animations, which are also possible, but something wholly different.
  • I won't talk further about keyframe animations in this course—you're welcome to explore them on your own.

Intro to DOM Programming

I'm going to expose the tiniest bit of JavaScript that should dovetail nicely with the concepts we’re becoming comfortable with. Namely, we know that when we load up a big HTML file that it it is converted into this very specific object, and this object, like any object, has properties that describe it. Some of the properties actually perform tasks, and as such, they are actually a function of the object, often called methods. Get comfortable with the terms property and method when talking about objects.

To further analyze this way of thought, picture your oven. It has knobs and burners, it has a color, a manufacturer, and it might have properties indicating its state, like clean or dirty. These are all properties, but let's note the differences between how they describe the oven. The color property does not change, the clean property will likely change values now and then. The knobs property is interesting; notice that they perform an action. Further, ‘acting’ on a knob might would likely change the temperature property.

As meta as this all seems, this abstraction perfectly describes programming. Looking back, we alread know we have a document object that it has properties and properties tell us things about the object, like when it was last modified: type document.lastModified into your browser console and see what happens (in fact, do it several times).

Of present concern is the things we can ask the document to do for us, like find elements. One of the most important parnts of a document is the fact that it can help us find other elements using query selectors—yes, the very same selectors you've been learning about for weeks!

Arguably the most useful feature of JavaScript is the tight integration to the Document Object Model, and the quickest introduction we can make is by employing our CSS techniques. The document object itself has many methods for selecting Elements, but perhaps the only you need to remember are the following two methods that belonging to the document object. Remember you can use the ‘.’ to access properties and methods, and the two methods in question are querySelector and querySelectorAll. These two methods can be fed regular css query selectors as arguments to find html elements.

document.querySelector(query)

Select and return the first node that matches the query
ex: document.querySelector('div') selects the first <div> in the document
ex: document.querySelector('div#test') selects <div id="test">

document.querySelectorAll(query)

ex: document.querySelectorAll('div') selects all <div>'s the document
ex: document.querySelectorAll('section > p') selects all <p>'s that are the first child of a section
  1. Open your console
  2. place your cursor after the > prompt
  3. If you havent yet, type document and inspect output
  4. type document.querySelector
  5. Output indicates function has not been called!
  6. type document.querySelector('h2');
  7. type document.querySelectorAll('h2');
  8. type document.querySelectorAll('h2').item(0);

Note, some people still use things like document.getElementById and document.getElementsByTagName, and you'll surely run into decades of tutorials suggesting you do the same. Be aware, modern methods provide a significant upgrade and I suggest you use the new stuff

CSS tricks Has an intro

Now that we're this far along, I'm attaching the full Selector whitepaper

:first-of-type
A pseudo-selector that selects the first element of a particular type within the context
section p:first-of-type selects the first paragraph of a section, ignoring whether or not it is the very first child.
:nth-of-type()
Can be used to select any element based on integer, even/odd, and cycle size (advanced).
li:nth-of-type(odd) {background-color:#EEE;} sets the background-color on the odd items of a list
:not()
A powerful technique I find invaluable, this pseudo selector negates a selector.
ex: p:not(.special) {...}selects all paragraphs that do not have a class of special.
ex: section p:not(:first-of-type) {...}selects all paragraphs of a section except the first one.
:before & :after
These pseudo-elements are a nice trick to add custom bullets, icons, or bits of boiler plate text to certain elements.
p:after {content: '...'} would add ‘...’ after of every paragraph.
abbr:before {content: attr(title)} would write the title attribute before each abbreviation.
:first-letter and :first-line
Another pair of pseudo elements that are somewhat self explanatory.
p:first-letter {font-size:500%;} would make the first letter of every paragraph 500% larger than the adjacent text.
p:first-line {font-weight: 800} would make the first line of text in every paragraph bold.
