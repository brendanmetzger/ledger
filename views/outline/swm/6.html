Transitions and Transforms

First, other than the initial 5 letters, transitions and transforms have little to do with one another. The reason they might be good to learn at the same time is that potential confusion could arise if you happened to let google auto-suggest take the reins.

From the W3C spec regarding transitions: CSS transitions allow property changes in CSS values to occur smoothly over a specified duration Essentially, transitions are animating from one style to another—which used to be the purview of fairly complicated JavaScript.

You'll notice a lot of resources mention vendor-prefixes. At this point, un-prefixed use of the transition property has wide support. Any tutorial or literature using a prefix is somewhat outdated. This is why it is so important to research and cite multiple resources for information.

Transforms, on the other hand, apply methods and vernacular commonly seen in programming, especially in regards to how elements are painted or drawn. From the spec on transforms: CSS transforms allows elements styled with CSS to be transformed in two-dimensional or three-dimensional space.

Keep in Mind

When you learn how to jazz things up it quickly becomes tempting to go a little crazy. Starting out, I say what the hell, go over the top... but I would be remiss if I didn't point out that sometimes effects can get annoying when the pile up or make things difficult to use. In addition to that, I've made a list of a few pointers for the road, if you will.

  • Transitions enhance presentation, they should not provide meaning.
  • Transitions degrade gracefully by default (or they should); ie, nobody should be impacted if they go missing.
  • Not all properties are animatable (display, for instance); here is the list of animatable properties.
  • Transitions are not Animations, which are also possible, but something wholly different.
  • I won't talk further about keyframe animations in this course—you're welcome to explore them on your own.

When using transforms, be aware that you'll only be able to apply transformations to a transformable elements, that is, an element that applies block-formatting via the box-model. Remember, any element can be forced into a block formatted element explicitly by setting display property to 'block' or 'inline-block', or implicitly when you have set a float or used an absolute or fixed position.
