Transitions and Transforms

Other than the initial five letters, transitions and transforms have little to do with one another. The reason they might be good to learn at the same time is that potential confusion could arise if you happened to let google auto-suggest take the reins; How do I... CSS... Trans... could send you down the wrong path.

From the W3C spec regarding transitions: CSS transitions allow property changes in CSS values to occur smoothly over a specified duration Essentially, transitions are animating from one style to another—which used to be the purview of fairly complicated JavaScript and is now, remarkably easy to accomplish.

When researching transitions, you’ll no doubt notice a lot of resources using vendor-prefixes. At this point, un-prefixed use of the transition property has wide support, and any tutorial or literature using a prefix is somewhat outdated. This is why it is so important to research and cite multiple resources for information.

Transforms, on the other hand, apply methods and vernacular commonly seen in programming, especially in regards to how elements are painted or drawn. From the spec on transforms: CSS transforms allows elements styled with CSS to be transformed in two-dimensional or three-dimensional space.

Keep in Mind

When you learn how to jazz things up it quickly becomes tempting to go a little crazy. Starting out, I say what the hell, go over the top... but I would be remiss if I didn’t point out that sometimes effects can get annoying when they pile up or make things difficult to use. In addition to that, I've made a list of a few pointers for the road, if you will.

  • Transitions enhance presentation, they should not provide meaning.
  • Transitions degrade gracefully by default (or they should); ie, nobody should be impacted if they go missing.
  • Not all properties are animatable (display, for instance); here is the list of animatable properties.
  • Transitions are not Animations, which are also possible, but something wholly different.
  • I won’t talk further about keyframe animations in this course—you’re welcome to explore them on your own.

More on Attributes

Selectors

Attribute selectors offer us yet another dimension of targeting Elements. I have made an example here, but essentially you use square brackets to inspect the presence, or even the value of a specified attribute. It will be important to understand the new syntax presented, as it is a good introduction to the style of syntax we use to execute programs.

For some reading material, here are the docs on attribute selectors, or for a more visual explanation check out this tutorial. Attribute selectors pretty much round out our selection suite, which all together, looks something like this: Element & descendant selectors, classes and IDs, children and sibling combinators, class chaining, and now attribute selectors. Both css and JavaScript employ these selection techniques.

Data

HTML has a way to store data in an attribute. Interestingly, it’s called a data attribute. Data attributes (well, all attributes) can be seen by the inspecting eyes of css, which is an introduction to attribute selectors. Likewise, data-attributes can be seen and set in very straight forward ways using object access methods (more on that to come), and this can be quite elegant. Within your html, you can simply add <p data-foo="bar"> where foo is the name of your custom attribute and bar is the value.

The important takeaway here is twofold. You can make up an attribute to store data for later (hint, JavaScript), but you can also target that attribute with the a css selector.

Intro to DOM Programming

I’m going to expose the tiniest bit of JavaScript that should dovetail nicely with the concepts we’re becoming comfortable with. Namely, we know that when we load up a big HTML file that it it is converted into this very specific object, and this object, like any object, has properties that describe it. Some of the properties actually perform tasks, and as such, they are actually a function of the object, often called methods. Get comfortable with the terms property and method when talking about objects.

To further analyze this way of thought, picture your oven. It has knobs and burners, it has a color, a manufacturer, and it might have properties indicating its state, like ‘is clean’. These are all properties, but let’s note the differences between how they describe the oven. The color property is static, that is it will not change, while the ‘is clean’ property may change as the result of other actions. The knobs property is interesting; notice that they perform an action—turning knob is expected to cause a change the temperature property.

As meta as this all seems, this abstraction is important, and perfectly describes programming. Looking back, we already know we have a document object that it has properties and properties tell us things about the object, like when it was last modified: type document.lastModified into your browser console and see what happens (in fact, do it several times).

Of present concern is the things we can ask the document to do for us, like find elements. One of the most important parnts of a document is the fact that it can help us find other elements using query selectors—yes, the very same selectors you've been learning about for weeks!

Arguably the most useful feature of JavaScript is the tight integration to the Document Object Model, and the quickest introduction we can make is by employing our CSS selector knowledge. The document object itself has many methods for selecting Elements, but there are really only two you need to remember. The two methods in question are querySelector and querySelectorAll, and you access them by using the ‘.’ operator on the document. These two methods can be fed regular css query selectors as arguments and will return the appropriate html element(s).

document.querySelector(query)
Select and return the first node that matches the query
ex: document.querySelector('div') selects the first <div> in the document
ex: document.querySelector('div#test') selects a <div id="test"> element.
document.querySelectorAll(query)
ex: document.querySelectorAll('div') selects all <div>'s the document
ex: document.querySelectorAll('section > p') selects all <p>'s that are the immediate children of a section.
  1. Open your console so you can query this document
  2. place your cursor after the > prompt
  3. If you haven’t yet, type document and inspect output
  4. type document.querySelector
  5. Output indicates function has not been called!
  6. type document.querySelector('h2');
  7. type document.querySelectorAll('h2');
  8. type document.querySelectorAll('h2').item(0);
  9. Compare output of #5 to #7

Note, some people still use things like document.getElementById and document.getElementsByTagName, and you’ll surely run into decades of tutorials suggesting you do the same. Be aware, modern methods provide a significant upgrade and I suggest you use the new stuff
