Notions of Functional Programming

For a long time the prevailing interest has been in Object Oriented programming, but with the rise in popularity of JavaScript and the ability to execute event-based and asynchronous applications, functional programming is becoming more and more popular.

First Class and Higher-order Functions

We already know all about these: a first order function can take another function as an argument, and return them as results.They are called first-class because the language allows them to be used where any other value is—like a number. Recall methods like addEventListener and setTimeout need callbacks to function correctly, thus we see immediately how important and fundamental to the JavaScript these concepts are.

Pure Functions

A general-use programming topic that may become useful down the road, pure functions are something worth understanding (if only the idea).

Will a function always return the same value?

Notice the two functions on the left. The multiply function is impure because it relies on variable state set elsewhere in the program. That variable can be changed elsewhere, thus, the reliability of the function itself is not guaranteed to return the same value every time.

Debugging can take up more time than necessary due to inconsistencies like these, but just as importantly, when we consider asynchrony or programs that can execute in multiple threads, the pure-ness of functions can ensure composition and thread-safety. This was perhaps not a huge deal in the past, but with the popularity of node.js and the advent of web workers, these ideas are necessarily less foreign.

Recursion

Iteration in a functional language is usually accomplished via recursion. A function that will invoke itself will perform that operation over and over until it has a base condition met; this sounds very familiar to the looping construct. Recursion has not typically been a suggested practice in javascript due to a condition known as stack overflow, referring to the depth of the call stack—or a function calling a function calling a function and so on. Typically you are only allowed to go so deep, depending on the compiler.

Map, Reduce, Filter

These functions were introduced in ECMAScript 5 and I consider indispensable. All three work on arrays; map returns a new array of the same length, each index having been operated on; reduce reduces an array to one value (sums, for instance), and filter returns a new array given that each item meets some condition.
