Types and Data Structures

Primative types in JavaScript are the building blocks of a program. Data Structures are assemblies of primitive types that represent a complex dataset or system. Imagining the human body as dataset, the individual organs are primative types. On their own they serve a specific function, but assembled into a whole structure they provide function and meaning.

Primative Types

Number
A number is a numeric value, represented as either an integer or a floating point numebr
Numbers are used with arithmetic operators such as +, -, *, /, and %
Invoking a method on a numeric type allows it to behave like an object, ie: (3.14159).toFixed(2); // 3.14
String
A string is a sequence of text that is surrounded by either single or double quotes: "This is a string"
Strings can be concatenated with the familiar + operator. 'I love ' + 'my cat' + ' (and dog)';
Invoking a method on a string type allows it to behave like an object "hello world".toUpperCase(); // HELLO WORLD
Boolean
Boolean values are either true or false. That's it.
Boolean values are often the results of expressions against logical or comparison operators
comparison example: console.log(5 > 6); // false, logical example: console.log(true && false); // false
null and undefined

example of null: var my_var = null; console.log(my_var === null); // true

Data Structures

Arrays
Arrays manage indexed data of any type in the structure of a list.
Arrays are always accessed via an integer that indicates the index. Indices start at 0.
Arrays can push, pop, shift, unshift (not specific to JavaScript).

Functions, Parameters and Scope

Functions are probably the most profound aspect of JavaScript programming, and from a very technical perspective, they are really Callable Objects. Just like an object, they have properties and values, and just like an object, those values may be of any type, including other functions!

Declaring a function

Functions can be declared by assignment or by using a named function. They have virtually the same outcome aside from a notion of hoisting, which you can forget about for awhile as it has few applicable purposes. Generally, it is recommended to use named functions, but the need for doing so is advanced and obscure, and again, can be glossed over at this stage.
