Warming Up

Today, start with writing a version of FizzBuzz found at the end of Program Structure in Eloquent JavaScript. I have witnessed many many solutions to this over the years and read dissertations from holier-than-thou types, but ultimately, this is a beautiful thinking problem that someone should just think about. FizzBuzz has a technical component, but if you fully read about it you start to understand that solving a problem is an act of an expression, not an act of locking in the solution. Perhaps it is why some individuals choose an art school. Perhaps that is why I've cumulatively spent about several hours playing with this problem when good programmers think it should take less than five minutes.

People feel anxiety when they sit down for math and programming because we are taught that smart people figure things out right away, when that is flat-out wrong. Smart people think about things, and sometimes it takes them years to think of something really really simple that someone else saw right away. Smart people think, and that is something that anyone can set aside a certain amount of time for.

I am an existentialist—both philosophically and computationally. I write my programs, starting from questions like what do I have? and what do I need?—very rarely do I use actual loops or conditions, and sometimes, just for fun, I will spend two hours rewriting a simple program to avoid them. I did that for my version of of FizzBuzz. In doing so, I unintentionally solved FizzBuzzWoof—and it felt good.

Matching Patterns and Counting in with StyleSheets

Brushing up on some patterns and light maths, briefly revisit some css properties and selection techniques.

Pseudo Classes—those special classes with a colon—allow you to target an element’s state or context. State means things like :focus or :hover, whereas context might mean an element’s position in the document. In bygone eras, developers would create classes to do what could often be described by an index number. For example, if you wanted the nth item to be bold, create a .nth class and then tack the attribute to the element in question. This was annoying for many reasons—especially if you are creating content on the fly with a feed or from some other origin than a html file that you have access to.

Today one has the availability of pseudo-classes like nth-child that allow you to select any element based on its position or index, or if it is even or odd. Better yet, it allows you to target any element that fits a pattern—like find every third element.
