Drawing and Animation

I've arbitrarily picked drawing and animation as an important topic because really, it presents an opportunity to learn both new material and hone already-learned skills. Truthfully, working with SVG and canvas can be a bit difficult; concerning the former, it can be especially difficult without a framework.

While you can draw with SVG or canvas, it will be worth understanding the difference between the two. Fundamentally, SVG is a vector format, canvas is raster, which isn't that important, but I will propose you start with SVG for a few other reasons. First, SVG is a file format, so you can use a drawing program such as Adobe Illustrator or Inkscape to draw, save, and embed files as images. Second, the file format is actually XML, and it's important to underscore how interesting that is. XML can be embedded directly into your document, meaning it becomes a part of the Document Object Model. You can query elements with JavaScript, and style elements with CSS, which is extremely useful. For these reasons alone, I think it's a great starting point. Canvas starts to outshine when you are embedding other raster and time based media (images and video), and applying filters and intense scene rendering and calculation—essentially games.

Jumping right in and writing all of your own SVG can be daunting—there are issues with both intense XML and as well as a foreign drawing API to contend with. For that, I will suggest using a framework, such as Snap, as a starting point to working with SVG. This is both good on how to read an API and learning about drawing in 2d cartesian space.

Before you begin with that, you must understand a little bit about svg itself. Here is a tutorial from the Mozilla Developer Network—I'd suggest starting with the introduction and reading up through Fills and Strokes. Here is another tutorial called Up and Running with SVG—I think browsing up through chapter 12 is relevant material. Here is the W3C SVG tutorial.

There is a glut of material on Lynda.com regarding SVG, with topics ranging from Adobe Illustrator to responsive design and of course programming. Browse that material to see if it interests you (especially while you still have access).
