Drafts

This directory serves no real purpose other than to highlight a practice you should warm up to, and that is versioning your work, refactoring it incrementally, and saving EVERYTHING. Often we are compelled to pile things up in one spot, afraid to delete anything because we are unsure, and commenting out bits and pieces in case we want to come back to something. Professionals use Version Control Systems like Git (GitHub) and Subversion so one can delete confidently knowing they have infinite redo if necessary. As usage of such tools falls well outside the realm of this course I’m leaving it up to you to create a system that works.

Software Versioning is a science unto itself. Whether you like numbers and decimal systems (1.0.1, 1.1.2) or Greek letters (alpha, beta), it really doesn't matter except that you backup you work and become quite aware of how you have your material organized. Generally, don’t throw anything away if you can tuck it away somewhere–code and files are cheap to store.
