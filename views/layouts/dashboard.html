Cache Invalidation

You may notice a random string of numbers preceded by a question mark when you link to your page—let me explain. The default behavior of a server is to do something as fast as possible, and depending on what time a file is requested it may respond with a special header code called a 304 Not Modified, in essence, telling your browser, load the same thing if you want, and a browser usually does. The server your work is hosted on is configured to send 304s quite liberally—generally ok to do for production sites—but it makes it difficult to develop websites over the internet.

Thus, when developing, you should (1) have dev tools open, and (2) then make sure cache is disabled (show note) to ensure you get the resource as intended. To make sure that you see you changes as requested, I append a harmless string of numbers (a common trick of cache invalidation) after the url that makes it look like a different resource to caching algorithims. This is called a query string, and they are harmless in most contexts and only used to avail in server-side programming anyway.
