Case Study: A Lightbox and CSS

Css is needed to create an overlay. I’m going to show you one way, but there are many facets you need to begin to understand. Before we begin the discussion, I want you to investigate the following topics on your own: css positioning, z-index, overflow, the vh and vw units. Before we begin our work together I may ask you to highlight and share something you learned.

What is an overlay in principle
it’s just a block level element, positioned in such a way that it overlays the entire page contents
It can cover the entire page, top to bottom, or just the visible screen 100vh and 100vw.
It will utilize position: absolute and some sort of z-index: (int)
The background can/should be locked
If the overlay fully covers page content, you don’t need to worry about scrolling (although it might be annoying)
If the overlay is only covering the viewport, you will need to disable scrolling
The background itself
There are many CSS coloring techniques that support an alpha channel (providing opacity).
Understanding Hierarchy, Show/Hide
The overlay layer is by definition a child of the body. It must exist but only be visible when needed.
Controlling it’s visibility is something you need to spend some time thinking about, and there are several css properties to consider: opacity, display, visibility, height and width
Here’s an important tip: you can you JavaScript to read/write styles directly on a dom element to control visibility, you should strongly consider using classes to do this. If you want to hide an element, where should the class be applied?
