Anatomy of the cascading stylesheet language

There are essentially two things that make a stylesheet work, the first is the selector, and once the selector has found its target, you can begin modify the targets properties. Become familiar with those two terms, I will say selector and property with some frequency. Selector means find this node and properties means adjust the way this thing looks.

Selectors

Selectors are the handles that target specific HTML Element Nodes. Specific means anything from one element such as #special { …props to literally every element, with something like * { …props. Selectors can be concatenated with a comma to apply one set of property settings to several selectors—concatenating the previous examples would look like so: #special, * { …props. Selectors are a heady topic, and one that we will be spending easily the next five weeks on.

Blocks

A block, as far as stylesheets are concerned, is a group of properties. In fact, its worth getting comfortable with seeing { curly braces } all over the place, and recognizing that you never, ever, (ever) see a bracket delimiter all by itself. If it was opened, it must be closed. One ore more selectors are followed by a block, which represents how the selected elements will be styled.

Syntactic Concerns

Stylesheet languages, like Markup Languages, are declarative: you say what you want to happen, and some other program determines how that happens. Declarative languages can be quite forgiving, which is mostly a good thing. Good in that you talk about what you want to do, not how it is to be done. They are flexible, and generally if the parser does not understand an instruction, it simply skips it. Here is where we get a taste of the bad; a genuine f-up in a declarative language can lead to confusion if let fester. Try your best to be organized, and as always, have the console open (every year tools get better at helping point out your mistakes). Glancing at an overview of the syntax, the declaration of a CSS ruleset resembles—at least typographically—a programming data structure. Curly braces, colons, semicolons and careful use of punctuation operators is the starting point.
