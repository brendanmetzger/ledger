Functions & Pseudo Selectors

The concepts of ‘pseudo’ classes and elements get us ever closer to designing abstractly.

The application of pseudo classes to apply styles depending on order of elements with selectors utilizing :nth-child is conclusive step forward to thinking about design in terms of context. Now comes the introduction of the limited ways that css can generate a tiny bit of content on its own, how to utilize variables in stylesheets, and the ability of the calc function to do some interesting tricks. These principles are a gateway to writing optimized software utilizing principles such as dry.

A significant portion of interactive design—as well as programming—requires us to look at design and development fundamentally as calculations and conditions. Designing this way is abstract and once learned, significantly more powerful than one-dimensional stuff. No longer would we write something like make this 20px wide, but rather make this 1/5 of the viewport, where the viewport is largely unknown in all circumstances. The reason we develop a vocabulary of selectors is so that we can loosely couple html structure from css view considerations, and likewise, from JavaScript. With loose coupling, you can make changes to any facet of your project without affecting the other areas. We will need to learn a little bit about functions—which can be thought of as machines that takes input to modify, perform operations, and usually return output.

css variables
css variables are a relatively new concept that allows you to specify a setting once, and use it later in your code. Variables are useful whenever you are repeating information, thus you only have to specify it once and change it once if necessary.
Be sure to keep track of the support tables when you are doing work that requires cross-browser compatibility.
:root
The root :root of your document is the <html> element. While you could certainly use a type selector, it can easily be overridden*, so the :root pseudo class is a good place to store important things, like variable declarations.
* overriding style declarations is easy and useful—which is why first letter in CSS stands for cascading. However, without understanding that principle, confusion often arises when you seem to be making a change in your code but don’t see that change in your browser.
the calc function
The calc function allows you to multiply, add, subtract, and divide numbers to create new dimensions—and the numbers can be of any unit.
Useful for setting a minimum constraint such as type size, ie., font-size: calc(3vmin + 1em)
useful for obscure column widths, ie., width: calc(100% / 9)
The negation pseudo class :not
This pseudo class negates any selector inside the argument.
example: p:not(.special) selects all paragraphs that do not have a class of special.
example: section p:not(:first-of-type) selects all paragraphs of a section except the first one.
example: article > *:not(section) selects every child of an article that is not a section.
:before & :after
These pseudo-elements are a nice trick to add custom bullets, icons, or bits of boiler plate text to certain elements like quotation marks surrounding text enclosed with <q> Elements (revisit this example)
p:after {content: '…'} would add ‘…’ after of every paragraph.
abbr:before {content: attr(title)} would write the title attribute before each abbreviation.
The attr function
Another function that allows you to use the contents of an attribute in a CSS property. As of this writing, you can only affect the content property, but that will very likely change in the coming years. One such feature would be for a tooltip (example here).
:first-letter & :first-line
Another pair of pseudo elements that are somewhat self explanatory.
p:first-letter {font-size:500%;} would make the first letter of every paragraph 500% larger than the adjacent text.
p:first-line {font-weight: 800} would make the first line of text in every paragraph bold.
