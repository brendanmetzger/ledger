css combinators

We are familiar with descendant selectors, something like article section will allow us to style all any section so long as it has an article as an ancestor. Much of the time, this can be more than we want, but luckily we can limit the reach of the descendant chain with the use of a Child Selector. By simply changing our previous selector to article > section we only select sections that are direct children of an article. If you’re not familiar with the difference between children and descendants yet, hit the books.

While not as immediately useful, adjacent sibling selectors can be utilized to apply classes based on if elements are directly next to each other. An example would be adding a top border all but the first element of a list with li + li { border-top: 1px solid #000; }. If say, you wanted all the paragraphs following the first one to have an indent, but couldn’t be sure that it would be only p tags, you could switch to general sibling selectors and use a selector like p ~ p { text-indent: 1em; }.
