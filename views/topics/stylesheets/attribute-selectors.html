Selecting by Attribute

Selectors

Attribute selectors offer us yet another dimension of targeting Elements. Essentially you use square brackets to inspect the presence, or even the value of a specified attribute. It will be important to understand the new syntax presented, as it is a good introduction to the style of syntax we use to execute programs.

For some reading material, here are the docs on attribute selectors, or for a more visual explanation check out this tutorial. Attribute selectors pretty much round out our selection suite, which all together, looks something like this: Element & descendant selectors, classes and IDs, children and sibling combinators, class chaining, and now attribute selectors. Both css and JavaScript employ these selection techniques.

Data

HTML has a way to store data in an attribute. Interestingly, itâ€™s called a data attribute. Data attributes (well, all attributes) can be seen by the inspecting eyes of css, which is an introduction to attribute selectors. Likewise, data-attributes can be seen and set in very straight forward ways using object access methods (more on that to come), and this can be quite elegant. Within your html, you can simply add <p data-foo="bar"> where foo is the name of your custom attribute and bar is the value.

The important takeaway here is twofold. You can make up an attribute to store data for later (hint, JavaScript), but you can also target that attribute with the a css selector.
