Everything is a box

Depending on your major, your perspective on web design/development from that of your classmates my be quite stark. It is likely that designers in this class first learned layout from a graphic perspective, attempting to apply design principles to their HTML and css work. Students with a programming background may have learned to apply 3rd party frameworks to solve problems consistently or efficiently, perhaps using html and css as a means to an end. Thus, both groups have an achilles heel; when looking for a shortcut, designers avoid the rigor of programming while programmers avoid the rigor of design. The good news for designers is that programming is very much about design, and the good news for programmers is that good design is very much about precision and optimization.

The Box Model

Conceptually, it has been around long enough to have many many tutorials and write-ups describing every last detail. The nutshell is adjacent: the area in yellow is the margin, green is the padding, gray is the border and blue is the content. Within these constraints you organize the position and layout of content. Understand that you can set the top, right, bottom, and left margin, padding and border independently.

A decade ago the box-model was implemented differently in different browsers (and it was a nightmare). Perhaps the bitterness of the past leaves me confident to conclude that with only one set of straightforward rules, it you should spend a few hours learning what there is to learn about it. That’s right, a few hours, and there you are, you've learned it!

Box-Sizing

The box-sizing property Box sizing is a newer feature, and I'll briefly say that while the box model can be computed in two ways–either content-box (default) or border-box, it’s generally recommended to set everything to border-box once you know the difference. You can set a property to all elements with a universal selector.

Positioning

You have probably noticed by now the crosshairs following you around this page. In the console, you will notice that the position of the crosshair is logged upon every move. Study the log and note that our browser viewport is governed by the Cartesian coordinate system. To pinpoint something in space, you start with the origin, and in they layout/box-model world, the origin is typically the top and left of the box or viewport, but you can also use bottom and right. Now that we understand units, and have a gist of the box model, we can talk about the position property. The big ones we will concern ourselves with are static (the default), relative, absolute and fixed.

Setting the position should not be confused with setting the margin, padding, and border, and is best utilized to make dramatic, finite changes. For some perspective, in a large application, I will use margin and padding on almost every element, while I may only set the position of an element a handful of times.

Layout & Floats

Learning layout is obviously quite important to learning how to apply a design. With the proliferation of tablets and mobile devices, the toolset for controlling layout is moving very quickly. I'd advise you to go through this tutorial called Learn CSS Layout (it will also be assigned in the homework). Advanced students will note at the end the introduction of flexbox and probably want to investigate those techniques. I won’t be working flexbox into the curriculum; at the moment the spec isn’t entirely consistent and I’m admittedly confused about best practices and fallbacks at this point. it’s fairly safe to say, however, that this will probably be the way we do layout in the future, so take that to heart.
