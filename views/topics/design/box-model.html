Depending on your major, your perspective on web design/development from that of your classmates my be quite stark. It is likely that designers in this class first learned layout from a graphic perspective, attempting to apply design principles to their HTML and css work. Students with a programming background may have learned to apply 3rd party frameworks to solve problems consistently or efficiently, perhaps using html and css as a means to an end. Thus, both groups have an achilles heel; when looking for a shortcut, designers avoid the rigor of programming while programmers avoid the rigor of design. The good news for designers is that programming is very much about design, and the good news for programmers is that good design is very much about precision and optimization.

The Box Model

Conceptually, it has been around long enough to have many many tutorials and write-ups describing every last detail. The nutshell is adjacent: the area in yellow is the margin, green is the padding, gray is the border and blue is the content. Within these constraints you organize the position and layout of content. Understand that you can set the top, right, bottom, and left margin, padding and border independently.

A decade ago the box-model was implemented differently in different browsers (and it was a nightmare). Perhaps the bitterness of the past leaves me confident to conclude that with only one set of straightforward rules, it you should spend a few hours learning what there is to learn about it. That’s right, a few hours, and there you are, you've learned it!

Box-Sizing

The box-sizing property Box sizing is a newer feature, and I'll briefly say that while the box model can be computed in two ways–either content-box (default) or border-box, it’s generally recommended to set everything to border-box once you know the difference. You can set a property to all elements with a universal selector.

Positioning

A browser viewport is governed by a version of the Cartesian coordinate system. To pinpoint something in space, you start with the origin, which in the browser world, is the top and left of the viewport ( but you can also use bottom and right to position elements). Now that we understand units, and have a gist of the box model, we can talk about the position property. The big ones we will concern ourselves with are static (the default), relative, absolute and fixed.

Setting the position should not be confused with setting the margin, padding, and border, and is best utilized to (1) move something to a specific point in space and (2) pull an object out of the flow of the document (perhaps to overlap elements), often both in combination, such as a light box overlay for viewing media. * Setting the position to absolute or fixed pulls the element out of the document flow, see my note in the float example below.

Layout & Floats

Learning layout is obviously quite important to learning how to apply a design. With the proliferation of tablets and mobile devices, the toolset for controlling layout is moving very quickly. I'd advise you to go through this tutorial called Learn CSS Layout. Those who have done their research will note at the end the introduction of flexbox and probably want to investigate those techniques. Flexbox is tremendous, although a tad complicated so that will be covered in the SWM2 curriculum.

Watch the carousel below carefully as it attaches new classes. You may see a pattern emerge, but if not, keep at it, this may be one of the most confusing parts of css floats. Specifically, notice the containing element, how it collapses when items are floated. This is by design, but it takes a long time to understand when and precisely why this happens.

Whenever you you apply a float property to an element, that element is removed from the normal flow. This also happens when you apply absolute or fixed positioning. Essentially this means the the parent containing the element no longer knows about the size and shape of the floated content. Imagine a basket of tennis balls in the bottom of an empty swimming pool. Fill the pool with water, the nature of the relationship between the tennis balls and the basket changes significantly—notably the basket doesn’t do anything. We can get around this by implementing a clearfix (covered in the tutorial).
