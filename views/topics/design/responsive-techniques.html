Responsive Design

Responsive web design is a technique used to modify layout, typography and other design considerations in response to the size and type of device used by the viewer. It can be elusive and frustrating, so some philosophy is necessary.

Mobile First

Mobile-first design is an important concept to practice if you intend to make fully-responsive web sites or applications. Essentially, it means that you design something for the smallest screen you are supporting, and then you scale up to accommodate bigger screens and the more space you have available. The fact of the matter is that your mobile-first design is always very simple: it is only one column, type is generally designed to be read about 2 feet from a user’s face and interactive elements need to accommodate a fingertip.

While this seems like an obviously good choice, in practice, few designers (or clients) are patient enough for this process, for a couple of reasons. Often the mobile design is done after the desktop because then a client can approve and adjust the look and feel. In addition, most designers don’t work in an interactive design medium (like a text-editor/browser) so they are not very aware of the myriad elements that one can and should consider when making decisions.

From experience, these caveats delay a project significantly and it’s painful and awkward to go from full-scale design to small, but relatively easy to go small to big. In other words, put on your skinny jeans before your holiday meal, not after. So if you want to get the knack of mobile development, always start mobile first.

Media Queries

Media Queries are pretty simple—if you have the gist of CSS, the won’t give you any trouble. Essentially, imagine you can divide your styles up by certain criteria: if the screen is too small, load this stylesheet, and likewise, you can target device types, landscape vs. portrait, etc. Say you asked yourself, I want some really big text, but only on a really big screen.. Media Queries are where you begin, and once you get your site schematics figured out and media in place they can offer immediate solutions to problems with text size and layout problems. The W3Schools reference is also a good starting point.

Addenda

There is another technical item you should be aware of. Be sure to investigate mobile-appropriate meta tags to make sure your css is applied correctly. Remember, meta elements are used as instructions—they are invisible to users, but machines use that information to make other decisions.

Responsive design is a lot of philosophy and a little bit of technique. You've seen me employ all of the techniques you will ever need at this point: relative and viewport units, and now, media queries.
