Data & The <input> element

In this course, we will mostly be concerned with the basic use of forms to provide input and a point of interaction. It is worth briefly mentioning that they are actually quite complex, and necessarily so: the provide the interface to send huge amounts of information to a back-end server for storage, processing, payment and many other activities. For this reason, they have hundreds of attributes concerning data protection, validation, and accessibility. As such, they can be a bit overwhelming, so for a first pass, the introduction will be brief and any further interest can be self-directed. The Mozilla Developer Network has a nice guide for creating forms if you are interested.

Of fundamental concern to us as interactive designers and developers is a notion of input/output. We are quite familiar with the mouse/trackpad as an input device: x/y movement coordinates, clicking, selection and scrolling are executed through such devices. Considering the keyboard, we have the full range of a keystroke (down, up, and press) which can be executed on any key, and along with meta keys (option, command, control).

Inputs

Input is largely controlled via the input Element. The input is an empty element, and depending on the type attribute, it well exhibit a different function. For instance, <input type="text"… is your basic text input field, where <input type="checkbox"… would be a checkbox. You can make many types of input, below is a quick overview of some interesting ones.

<input type="text" value="content"/>
The text input is your go-to for basic input. The value attribute holds whatever is typed into the box
<label><input type="checkbox" value="mmm-hmmm"/>check?</label>
The checkbox is interesting: you set the value, and the value only is sent if the box is checked.
The Label Element provides additional functionality.
<input type="range" min="0" max="100" step="5" value="50"/>
<input type="color" value="EE33AA"/>
