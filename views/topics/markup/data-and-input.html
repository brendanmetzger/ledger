Data & The <input> element

For introductory purposes, one only needs a very basic implementation of forms to solicit input interactively. It is worth briefly mentioning that they are actually quite complex, and necessarily so: they provide the interface to send huge amounts of information to a back-end server for storage, processing, payment and many other activities. For this reason, they have hundreds of attributes concerning data protection, validation, accessibility and many other nuances. As such, they can be a bit overwhelming, so for a first pass, the introduction will be brief and any further interest can be self-directed. The Mozilla Developer Network has a nice guide for creating forms if you are interested.

Of fundamental concern to interactive designers and developers is the notion of input and output. The mouse/trackpad is a perhaps the most familiar with input device: x/y movement coordinates, clicking, selection and scrolling are executed through such devices. Considering the keyboard, we have the full range of a keystroke (down, up, and press) which can be executed on any key, and along with meta keys (option, command, control) which is also a powerful mechanism.

Inputs

Input is largely controlled via the input Element. The input is an empty element, and depending on the type attribute, it well exhibit a different function. For instance, <input type="text"… is your basic text input field, where <input type="checkbox"… would be a checkbox. You can make many types of input, below is a quick overview of some interesting ones.

<input type="text" value="content"/>
The text input is your go-to for basic input. The value attribute holds whatever is typed into the box
<label><input type="checkbox" value="mmm-hmmm"/>check?</label>
The checkbox is interesting: you set the value, and the value only is sent if the box is checked.
The Label Element provides additional functionality.

And More!

That is an extremely terse list; there are textareas, radio buttons, and many more types of input to do exactly what you intend. Knowing the inputs available and how to use them is an important part of interactive design and programming. Be sure to know about the form events available—in my examples above I am using the oninput events. You should understand what blur and focus mean, as well as using the form values.
