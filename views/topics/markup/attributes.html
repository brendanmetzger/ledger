Nodes Have Attributes

An attribute is a quality that helps categorize any object. For instance, most individual humans have a name. The value of the name is assigned together help define and represent the object. This could be codified say, with <human name="bob"/> where human is the object, name is a attribute, and bob is the value of it.

The purpose of attributes are to store information about objects, largely so that when you need to examine them, find them, or interact with them they have special characteristics that can be utilized. Sometimes attributes are mandatory like the href in anchors—without them an anchor wouldn't know where to link to—in this way the href exhibits behavioral control. Attributes like class can be used to describe elements so they can be located in css selector declarations—a class provides a method of identification.

Syntactically, an attribute is always written on the opening tag of an element and is defined by specified keyword (meaning you don't make them up), then an = sign and a quoted string of text which usually the author is responsible for providing or naming. Back to our example <a> (anchor) element, the href attribute (known as the hypertext reference) is a standard attribute and the value is supplied by the author as the resource to direct the user to.

Attributes are now an essential part of our vocabulary as we will be learning and using new ones throughout the semester. Attributes like href only apply to specific elements, while others are universal, in that we can apply them to any element. html even offers the facility to create your own attributes so long as they are prefixed with data- as in <p data-foo="bar">
