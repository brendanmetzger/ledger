Nodes and their Attributes

An attribute is a quality that helps categorize any object. For instance, most individual humans have a name. The value of that name, once assigned, helps to define and represent the object. This could be codified say, with human name="Oprah" where human is the object, name is an attribute, and Oprah is the value of it.

The purpose of attributes are to store information about objects, largely so that when you need to examine them, find them, or interact with them they have special characteristics that can be utilized. Sometimes attributes are mandatory like the href in anchors—without them an anchor wouldn't know where to link to—in this way the href exhibits behavioral control. Attributes like class can be used to describe elements so they can be located in css selector declarations—a class provides a method of identification. The id attribute is perhaps the most misunderstood, in that it is used loosely as as a selector in a stylesheet. Identification attributes, like a social security number, represent identity. This is an important topic—you do not necessarily need to locate someone by ssn when a combination of assets might work fine, like name and address.

Syntactically, an attribute is always written on the opening tag of an element and is defined by specified keyword (meaning you don't make them up), then an = sign and a quoted string of text which usually the author is responsible for providing or naming. Back to our example <a> (anchor) element, the href attribute (known as the hypertext reference) is a standard attribute and the value, supplied by the author, holds the location of a reference point (like another page).

Attributes are an essential part of the vocabulary for describing documents. Same attributes like href only apply to specific elements, while others are universal, in that we can apply them to any element. html even offers the facility to create your own attributes so long as they are prefixed with data- as in <p data-foo="bar">
