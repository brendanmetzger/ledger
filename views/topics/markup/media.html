Using file paths

Some folks are predisposed to hating paths, I would assume they dislike crossword puzzles, scrabble and word hunts. Even if such games go unappreciated, it would be hard to argue that they are conceptually difficult to wrap your head around. The same is true for paths‚Äîthey have a few rules and once you understand them, they are not difficult.

The Rules

  1. There are two special symbols: . means current directory and .. means parent directory
  2. The dots, if not at the very beginning of a path must be touching slashes‚Äîor said another way, should never be touching letters; ie., not..ok vs. yes/../ok vs ..no/way vs. ../good/job
  3. Moving directories is expressing a location: Understand that writing yes/../ok is precisely the same thing as saying ./ok. There are infinite ways to move around a filesystem if you want to be absurd about it
  4. A plain old / means directory and if used at the very beginning of a path (without a dot) it means go to the beginning or root of the filesystem. When used with dots and words, it means move in and out of particular directories
  5. Paths implicitly start from the the directory of the file they are specified in‚Äîyou can explicitly use dots and slashes to change to an explicit starting point.
  6. Words between slashes represent file names, and they must be an exact match
  7. Whitespace matters‚Äîtry picturing it as pink-space if you must; clearly dirüÅ¢/src is not the same as dir/src
  8. Blank spaces are hard to see, and a lot of mistakes are simply forgetting to notice it. Thus, I recommend never using non-word characters except for hyphens and underscores in file names.

Embedding Audio/video

Audio and video are only slightly more complicated. Depending on your format, you may have to supply multiple video types, but for the vast majority of users, an mp3 or m4a will work for audio or a mp4 for video.

    <audio src="valid/path.(mp3|m4a)" controls="controls">
      Your browser does not support the audio element.
    </audio>

    <video src="valid/path.(mp4|mov)" controls="controls" poster="valid/path..(jpg|png|gif|svg)">
