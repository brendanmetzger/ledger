Multimedia, Embedded and Scripted Content

This content can often be described as extraneous to the page it is embedded in—or said more plainly, the content often lives in a different file or a different format than standard html. Images are probably the most notable, but this includes other media (audio/video) as well as media generated by programs—such as JavaScript and the canvas element.
Element Description TL;DR
area The area element defines a hot-spot region on an image, and optionally associates it with a hypertext link. This element is used only within a map element. Avoid it.
audio The audio element is used to embed sound content in documents. It may contain one or more audio sources, represented using the src attribute or the source element; the browser will choose the most suitable one. Not only gives you a player, but also exposes a robust JavaScript api
img An img element represents an image and its fallback content. The image is specified in the src attribute. The value of the alt attribute and the content referred to by the longdesc attribute are the img element’s fallback content and provide equivalent content for users that cannot or will not process images. Crucial to understand the complexity of imbedding images.
map The map element is used with area elements to define an image map (a clickable link area). Continue Avoiding.
track The track element is used as a child of the media elements—audio and video. It lets you specify timed text tracks (or time-based data), for example to automatically handle subtitles. The tracks are formatted in WebVTT format (.vtt files) — Web Video Text Tracks. Should be used more often for adding subtitles, but that is often a service few small time producers can afford to implement.
video Use the video element to embed video content in a document by specifying a url in the src attribute. Not only gives you a player, but also exposes a robust JavaScript api
