Text Level Semantics

Use the HTML inline text semantic to define the meaning, structure, or style of a word, line, or any arbitrary piece of text. Here is the full whitepaper for review.
Element Description TL;DR
a The Anchor Element (a) defines a hyperlink to a location on the same page or any other page on the Web. It can also be used (in an obsolete way) to create an anchor point—a destination for hyperlinks within the content of a page, so that links aren’t limited to connecting simply to the top of a page. Perhaps the most important Element ever created, how you link from one place in on the internet to another.
abbr The abbr element (or HTML Abbreviation Element) represents an abbreviation and optionally provides a full description for it. If present, the title attribute must contain this full description and nothing else. Surprisingly useful—know this one.
b The b Elementrepresents a span of text stylistically different from normal text, without conveying any special importance or relevance. It is typically used for keywords in a summary, product names in a review, or other spans of text whose typical presentation would be boldfaced. Another example of its use is to mark the lead sentence of each paragraph of an article. 9 times out of 10 you should be using the strong Element—but if you just need to have bolder text with no semantic meaning, this is your pick.
bdi The bdi Element (or Bi-Directional Isolation Element) isolates a span of text that might be formatted in a different direction from other text outside it. Rarely need this in western text.
bdo The bdo Element (or HTML bidirectional override element) is used to override the current directionality of text. It causes the directionality of the characters to be ignored in favor of the specified directionality. Rarely need this in western text.
br/ The line break br/ produces a line break in text (carriage-return). It is useful for writing a poem or an address, where the division of lines is significant. 99.9% of the time this is misused to control padding. Do not use the br/ for that. Only application is for literally adding an arbitrary line-break to a sentence.
cite The Citation Element (cite) represents a reference to a creative work. It must include the title of a work or a URL reference, which may be in an abbreviated form according to the conventions used for the addition of citation metadata. Occasionally useful fore heavily marked up documents.
code The Code Element (code) represents a fragment of computer code. By default, it is displayed in the browser’s default monospace font. Renders a monospaced font by default. Depending on the nature of the explanation, quite handy.
data The data Element links a given content with a machine-readable translation. If the content is time- or date-related, the time must be used. I've honestly not used this yet, but I like to know I could.
dfn The Definition Element (dfn) represents the defining instance of a term. Will italicize text as its default style. Useful for heavily marked-up documents.
em The emphasis Element em marks text that has stress emphasis. The em element can be nested, with each level of nesting indicating a greater degree of emphasis. This element should be in heavy rotation—it also italicizes text.
i The i Element represents a range of text that is set off from the normal text for some reason, for example, technical terms, foreign language phrases, or fictional character thoughts. It is typically displayed in italic type. 9 out of 10 times you should be using a em Element. But if you need to italicize something for eyeballs only (no semantic meaning), this will do it.
kbd The Keyboard Input Element(kbd) represents user input and produces an inline element displayed in the browser’s default monospace font. Not widely used
mark The Mark Element (mark) represents highlighted text, i.e., a run of text marked for reference purpose, due to its relevance in a particular context. For example it can be used in a page showing search results to highlight every instance of the searched-for word. Picture what a highlighter does.
q The Quote Element(q) indicates that the enclosed text is a short inline quotation. This element is intended for short quotations that don’t require paragraph breaks; for long quotations use blockquote element. Quotations—and you can do even fancier things with quotation marks using css.
rp The rp Element is used to provide fall-back parenthesis for browsers non-supporting ruby annotations. Ruby annotations are for showing pronunciation of East Asian characters, like using Japanese furigana or Taiwainese bopomofo characters. The rp element is used in the case of lack of ruby element support its content has what should be displayed in order to indicate the presence of a ruby annotation, usually parentheses. Not useful in western text.
s The Strikethrough Element (s) renders text with a strikethrough, or a line through it. Use the s element to represent things that are no longer relevant or no longer accurate. However, s is not appropriate when indicating document edits; for that, use the del and ins elements, as appropriate. Crosses something out. Especially important to signify editorial changes to a document.
samp The samp element is an element intended to identify sample output from a computer program. It is usually displayed in the browser’s default monotype font (such as Lucida Console). Not used all that often.
small The Small Element (small) makes the text font size one size smaller (for example, from large to medium, or from small to x-small) down to the browser’s minimum font size. In HTML5, this element is repurposed to represent side-comments and small print, including copyright and legal text, independent of its styled presentation. Think legalese and stuff you don't think anyone would try to read. Not intended to control size of text.
span The span Element is a generic inline container for phrasing content, which does not inherently represent anything. It can be used to group elements for styling purposes (using the class or id attributes), or because they share attribute values, such as lang. Means nothing—so if you want to style something and there is literally not a more appropriate element to choose, this is what you turn to. You really need to understand that last sentence.
strong The Strong Element (strong) gives text strong importance, and is typically displayed in bold. Bolds texts and imparts the meaning of bold text—as in it is something important.
sub The Subscript Element(sub) defines a span of text that should be displayed, for typographic reasons, lower, and often smaller, than the main span of text. Subscript, the darling of math and science.
sup The Superscript Element (sup) defines a span of text that should be displayed, for typographic reasons, higher, and often smaller, than the main span of text. Superscript, also the darling of math and science.
time The time Element represents either a time on a 24-hour clock or a precise date in the Gregorian calendar (with optional time and timezone information). This one should be used more often—accurately representing dates and times is quite important (if you use them)
u The Underline Element (u) renders text with an underline, a line under the baseline of its content. Be careful with this—ux generally reserves underlined things to mean it is a link.
var The Variable Element (var) represents a variable in a mathematical expression or a programming context. Not used all that often outside of technical write-ups.
wbr The element word break opportunity wbr represents a position within text where the browser may optionally break a line, though its line-breaking rules would not otherwise create a break at that location. If you are very uptight about how you want a line to break, this is your jam. I've never used it myself.
