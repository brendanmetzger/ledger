Practice

Practice is where you make small gains on the necessary techniques. Practice is about fundamentals—like playing an instrument or learning the skills needed in sport—incremental gains eventually lead to mastery.

Grading

Practice is on you, and you must maintain a practice routine that accommodates your schedule. I have set up a cron task which will scan your work on the iam server every night like a search crawler. That means if you do some work, but forget to upload it, it will not be counted for practice that day.

I am only concerned with html, CSS and JavaScript I have provided you in the templates. After I find updates, I'll place them into a version control system to keep your changes organized. This type of software is designed to allow developers to track changes and view differences between files. In other words, do not pretend to practice for the points—it will be obvious and it tells a story about your work ethic that may be worse than doing nothing.

I have a function set up, much like an ease-in-out animation. One commit is worth almost nothing, but maintain three to four per week and you will be great shape.
