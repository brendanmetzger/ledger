Practice

Practice is where you make small gains on the necessary techniques. Practice is about fundamentals—like playing an instrument or learning the skills needed in sport—incremental gains eventually lead to mastery.

Grading

You must implement and adhere to a practice routine that accommodates your schedule. I have set up a cron task which will scan your work on the iam server every night like a search engine. That means if you do some work, but forget to upload it, it will not be counted for practice that day, no exceptions.

I am only concerned with html, CSS and JavaScript and the README I have provided you in the templates. If updates are found, those updates will be tracked in a version control system to keep your changes organized. This type of software is designed to allow developers to track changes and view differences between files. In other words, do not pretend to practice for the points—you will earn a poor critique for pretend work.

For every day you have done some work on a project a 'hit' will be recorded, up to 7 per week. I have a function set up, much like an ease-in-out animation. One hit is worth almost nothing, but maintaining three to four hits per week is where you need to be if you are concerned about grades.
