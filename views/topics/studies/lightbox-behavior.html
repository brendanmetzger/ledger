Case Study: A Lightbox and JavaScript

Recalling our case study from last week, an introduction to all of the important CSS css for making an overlay possible was given. Surprisingly, this is the harder half of the equation in terms of steps needed to solve the problem. As we understand that process, the remaining JavaScript is quite simple, and really only requires we understand our previously outlined material. Really, our case study continues thinking about several questions.

  1. When does a overlay need to be opened?
  2. Is it triggered by multiple elements?
  3. If triggered by multiple events, how should we assign a listener?
  4. How might we pay attention to if an overlay is open already, and what do we do with that information?
  5. How do we close the lightbox?

Much of the time you see <div onclick="…" is because it is easier, and you don’t have to concern yourself with page load, and this is something to start concerning yourself with. If you try to access the document object in your script file before everything has loaded, things don’t work quite right. But when does the page load? When questions are best answered through events, which are surveyed with listeners. Without further adieu: consider window.addEventListener('load', callback) and why it might be so crucial.
