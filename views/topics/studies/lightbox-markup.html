Case Study: Controlling overlapping content with CSS

Css is needed to create an overlay. I’m going to show you one way, but there are many facets you need to begin to understand. Before we begin the discussion, I want you to investigate the following topics on your own: css positioning, z-index, overflow, the vh and vw units.

What is an overlay in principle
It is a block-level element, positioned in such a way that it overlays the entire page’s body content
It can cover the entire page, top to bottom, or just the visible screen 100vh and 100vw
It will utilize the position: (absolute|fixed|relative) properties and use of the z-index: (int) property
The background can/should be locked
If the overlay fully covers page content, you don’t need to worry about scrolling (although it might be annoying)
If the overlay is only covering the viewport, you will need to disable scrolling using the overflow property
The background of the overlay element should be opaque
There are many CSS coloring techniques that support an alpha channel, namely rgba and hsla.
Utilize Hierarchy to implement show/hide scenarios
The overlay layer must be a child of the <body>, and while you need to create an element to hold the overlay content, it must only be visible when needed.
Controlling it’s visibility is something you need to spend some time thinking about, and there are several css properties to consider: opacity, display, visibility, height and width
Here is an important tip: you can you JavaScript to read/write styles directly on a dom element to control visibility, but you should strongly consider using classes to do this. If you want to hide an element, there are dozens of ways to do this and the most powerful way is via descendant and class selectors.
