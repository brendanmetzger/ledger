DOM Tricks

I've put together the following example not in the spirit of knowing it fully as an exercise, but as an anecdote to what interactive programming really entails; a lot of stylesheet manipulation, a a little bit of JavaScript and very thoughtful structure of content and data.

The :not pseudo class coupled :before pseudo element should read like the following: select h2 elements without a ‘cutout’ class and set some content before them.

Transitions are simple enough to implement, but designing with the full power of transitions can be quite masterful. In the example CSS I've provided, scan the document for <h2> Elements with a little pair of scissors before the text—click on them. Note in the code, the CSS is controlling the scissor icon with a pseudo element, and thus can be manipulated with CSS techniques we have available.

Look at the contrasting CSS and Javascript as it illustrates a key point about the nature of our programming languages. First, I haven’t changed any of my markup to accommodate the CSS or JavaScript I need to pull off this trick. The next is that CSS is by far most verbose part of any program I write and I've utilized complicated selectors to my advantage. The last bit is the JavaScript, which, when aided by efficient CSS and HTML, is typically concise.

Please Note: The JavaScript is obscure and I use some techniques I wouldn’t expect out of an advanced class, but this is only to illustrate this point. Try to get the gist first, and worry about the details later. You will learn easier techniques for accomplishing similar tasks.

When using transforms, be aware that you’ll only be able to apply transformations to a transformable elements, that is, an element that applies block-formatting via the box-model. Remember, any element can be forced into a block formatted element explicitly by setting display property to 'block' or 'inline-block', or implicitly when you have set a float or used an absolute or fixed position.
