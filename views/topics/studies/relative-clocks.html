The Regular Expression

Here is a Regular Expression (RegEx) to convert the string to a more consistent array. Reading from the left, the expression indicates: "match 1 or more digits, maybe a period and 0 or more digits followed by spaces and one of the following words: day, hour, minute or second" Mars' string of "24 hours 39 minutes and 35 seconds" converts to the Array ['24 hour', '39 minute', '35 second']

A doozy of a function

This is a function that takes a function as a callback parameter and returns a function. You may never quite wrap your head around this, but if you do, you will experience about 5 days of euphoria. Give or take.

Here is what it does: give it an integer for a start position, the total hours, and something to do—a callback—on each cycle. Hypothetically, say we had an alarm clock that needed to go off every cycle, starting at 8am, the code would be clock(8, 24, ⏰);

I've talked about it a little bit, so I’ll go over it a bit more: naming conventions are consistent use of variables to define what type of objects they hold. Referencing the adjacent code you will see the following conventions at play.

  • camelCaseStrings store functions, functions are executable
  • snake_case_strings represent primitive types, objects and arrays
  • FirstWordUppercase means that the object that is to be constructed with the new operator, like the Date object.
