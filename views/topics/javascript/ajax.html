A note on xml

I think xml is important, I think it is powerful and I think it is easy to use. Over the years, many of my colleagues have mocked my unwaivering dedication this format because every year they find a better way to pass data around, be it json or yaml or a hot new data structure yet-to-be (if you don’t pick up my sarcasm there, let me be clear, data formats are never hot or new). Here is a quick introduction to xml if it is completely foreign, otherwise let me highlight some of its key features.

  • xml applies the Document Object Model, verbatim
  • xml can store any data-structure and be converted to any other data structure
  • xml is human readable and self-describing
  • xml is used as the structure for several popular formats like pdf files and svg images
  • xml can be validated with a DocType (which is probably why the last two items mentioned use it)

Same Origin Policy

The thing that’s really difficult parts of ajax don’t have much to do with JavaScript, but with http. This protocol has loads of rules about headers, policy, request types and lots of abstruse mechanisms designed to make the internet reliable and functional.

If you've ever tried to do some quick ajax work, using jquery or something else, you may have found that it doesn’t work as planned. The fact of the matter is that to make an XHR request, you must be doing it over HTTP, which means if you haven’t set up a server on your own computer (localhost) you will have to develop exclusively on a server, such as iam.colum.edu. This all stems to the same origin policy, essentially a set of rules that enforce security and make it impossible to abuse the http protocol. In addition to only allowing files to be requested over http, files must match protocols, domains, and ports.

Because of this, I’m going to spend a little bit of time explaining the nuance of getting a server going and trying to get a simple request up and running. The bad news, this is surprisingly hard. The good news, once you’re past this, XMLHttpRequest is easy and invaluable. With that, let’s get started.

  • Use a text editor that starts a localhost, like Brackets
  • Install a package that runs a localhost, like Atom
  • start a localhost in a programming language like python python -m SimpleHTTPServer
  • or php php -S localhost:3000
  • Mac users can use Apache (already installed) and do all dev work there—recommended if you are going to invest in your workflow
  • Install an application like xampp that does a whole lot of developer stuff for you.
