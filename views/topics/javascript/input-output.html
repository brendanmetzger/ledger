I/O: Arguments and Return Values

One can go all day writing functions that expect no input and give no output—though a function that doesn’t do either is a bit rare. Functions like Math.random do not require input but it returns a random number between 0 and 1. A function like console.log takes any amount of input then performs a task without returning any value (which is why you see undefined when you run it in the console).

Arguments are named between the parentheses when you write the function—an when you call the function, you pass in a parameter. A return statement is placed at the end of a function, and it gives you something back. Just like some of the first mathematics we learned like cos x, a function takes some input and gives us some output. Unlike mathematics, functions in JavaScript don’t necessarily need input, and don’t necessarily provide output.
