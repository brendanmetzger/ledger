Lexical Scope

Recall for a moment, the beloved chestnut Whatever happens in Vegas, stays in vegas?. Now behold: Whatever variables are declared in a particular scope stay in that scope. Let’s explore.

Imagine an all-seeing, all knowing presence, out in the ether, beyond the multiverse. A presense so powerful it knows of the galaxies, stars, planets, corporate charters, sports teams, fauna, flora, fungi and bacteria it conjured into existence. It knows about them, how to talk to them, how to ask them things. Now imagine a universe precisely the opposite; reverse the order starting with the bacteria and ending with the stars, galaxies and the divine. This is the universe of lexical scope. To continue our metaphor, the bacteria, knows of itself and all the things that came before. It knows about all the things that make up its existence and it knows how to talk to them. Ending with the stars, galaxies, and the deity, it has all the answers. God, in this heretic metaphor, knows only what it sees around itself.

Now, before you we get too excited about our incredible godlike bacterium, let’s remember a major caveat of monotheism: gods don’t seem to care (or know) much about other gods, and bacteria is no different. The bacteria, all knowing as it believes itself to be, doesn’t know anything about the other bacteria it lives among, sharing it’s world

I’ll wrap it up even though we're delicately close to a philosophical breakthrough, let’s end the metaphor and say that scope is the nested structure of program. From deep within, scope means an area of a program that has access to the elements that were declared before, but not necessarily elements declared adjacent.

Closure

Closure is one of these things that takes awhile to grok but once you do, it isn’t nearly as profound as it seems.

  1. Any programming language will have rules that define scope.
  2. Scope is simply where variables are declared and what parts of a program can access them.
  3. A program always introduces something into a global scope. It is generally advised to keep this to a minimum*
  4. Arguments are variables; thus, arguments are scoped into the function they live in.
  5. Languages with first-class functions have closures.

* Frameworks are careful about polluting the global scope—this is why jQuery has the $, underscore has the _, etc; they are creating a namespace and wrapping all of their code behind it. No magic, just considerate authorship.

Global Scope

Everything we do in Javascript has a scope by default, the outermost called the global scope. If you create a new script, embedded or linked and start declaring variables, making expressions and statements, you are in the global scope. When we start declaring types such as objects and functions then variable declarations and expressions within the object and function, variables are local to that object; this is local scope
