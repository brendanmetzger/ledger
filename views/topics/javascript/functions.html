Functions; all the bits and pieces

Functions are probably the most profound aspect of JavaScript programming, and from a very technical perspective, they are really Callable Objects. Just like an object, they have properties and values, and just like an object, those values may be of any type, including other functions! Perhaps this sounds like nonsense now, but thus begins the introduction to JavaScript functions.

Declaring a function

I’m going to assume that for the most part we know what functions and why we make them, so instead, we’ll jump straight to mechanics. Functions can be declared by assignment or by using a named function. They have virtually the same outcome aside from a notion of hoisting, which you can forget about for awhile as it has few applicable purposes worth mentioning.

Regarding the first two examples, functions can be declared as above, or assigned as below. There is some advocacy for the former; declaring functions in this manner has some advantages with scope and hoisting. If that sounds daunting, don’t worry, it’s quite obscure and not worth getting into at this point. I advise you learn and use both as you see fit.

Looking at the next two, we see functions are values, and thus you can just declare them and do nothing. Or, you can just make an anonymous function and call it immediately–note those parentheses at the end. This is an immediately invoked function expression (IIFE). Framework authors use these constantly, and while they look intense, they are just a hop on our path to understanding functions.

Statements

First-Class Functions
Either accept a function as an argument or return an function as a value. This concept is extremely useful for code abstraction. JavaScript employs first class functions regularly.
Pure Functions
A function that does not mutate values or situations outside of the function itself. A higher level concept important to functional programming
Recursion
Algorithms that operate in an iterable fashion by evaluating only the input and whether or not it should execute itself again with the modified value of the input.
