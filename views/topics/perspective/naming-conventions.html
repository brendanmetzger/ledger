Naming Conventions

This is both the easiest thing to learn, also the hardest hardest, and worth revisiting often. As you get more familiar with writing source code, your efforts transcend from getting it to work into getting it to work in a way someone else can understand.

In programming, saying that hardly ever happens is just another way of saying it happens.

Douglas Crockford

Naming Conventions are a set of rules for choosing the character sequence to be used for identifiers, be it classes, IDs or variables in JavaScript. In HTML distinguishing between upper and lowercase is not extremely important for correct page rendering, but tag names should be in lowercase—that's a rule, not a recommendation. In CSS and JavaScript, upper and lowercase words are note the same. Whitespace is another obstacle—css classes and IDs cannot start with a number, and cannot contain any whitespace. In many ways, learning to program is simply learning to become extremely careful about typing.

This is something you generally learn over time, but standard cleverness/carefulness applies now. You can prefix and suffix IDs with common words to keep them unique but class-like. You can develop some taxonomy for CSS for common usage. In JavaScript—and really all programming—developers take this to another level. Not only is important to be consistent in your own work, there are many agreed upon conventions and techniques. Concerning JavaScript, browse these conventions and try to begin implementing some of the techniques.

While not particularly noteworthy, consistency is something that takes a lot of practice and effort. Try to keep your variable names, class names and any other name in a fashion where you (and others!) can make sense of what your code is doing.

snake_case

Snake case is a way of having multi-word strings by replacing white space with an underscore. In CSS, it's common to use a hyphen (-) as well.

camelCase

Camel Case is when you separate multi-word strings by replacing white space with the adjoining word converted to uppercase.
