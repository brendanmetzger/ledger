I use this as our starting example to programming because it illustrates a point that many neglect, which is programming is a series of small ideas that add up to massive opportunities. For starters, I think it is crucial to understand that counting is a way of describing situations. Examine the cmyk list above and ask/answer the following questions to yourself. Consider answering the questions using only numbers and universal symbols—as if to someone that knows arithmetic, but does not speak the same language as you. This is a simple concept and yet is very difficult.

  1. What is a list? Describe if you must, but ultimately, use only abstract symbols. This may be a sketch if you must.
  2. How long is the list? (A hint, the answer is a widely recognized symbol called an integer)
  3. Is there a first element in the list? How would you describe that in the context of the list, using only symbols?
  4. Is there an item that turns up last in the list? Describe it to me, using only symbols and operations?
  5. How would you describe the second (or third or fourth) to last item in the list, again, using only symbols?
  6. How would you move the first item to the last (assume that the list was in pencil on paper)?
