Coercion & Other Pitfalls

Type coercion can be a useful feature, or a really deceptive trick. Essentially, it is turning one type of variable into another: “7” and 7 are entirely different things, yet they can be the same if you want them to be… or if you aren’t paying enough attention.

10 % 5 & true
This code probably does what you think it will, in a way that is completely foreign. Using one ampersand is not an error, but a bitwise operation.
This is ever more insidious because & will coerce true into the number 0, and from there it gets really unpredictable. This is really the truest form of bug in that it won’t cause an error and even might work the way you intend (at first).
"" == 0
The equality operator, or == is not an identity operator. In a nutshell, the above definition is true
querySelector vs. querySelectorAll
These functions are very handy—pretty much all you have to remember to get any element out of the dom, but pay careful attention to the All at the end if you are trying to select a list of elements.
